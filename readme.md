## 다 읽고 메모는 하지 않은 책

- 객체지향 사실과 오해:
  - 코드가 없어서 잘 안 와닿는데 오브젝트 책으로 가는 방향인 것 같다.
- 함께자라기 애자일로 가는길:
  - 협업을 하면서 많은 도움이 된 것 같다.
- 소프트웨어 장인 정신:
  - 성장에 관련되어서 도움이 되었던 책 양치를 많이 한다고 양치의 고수가 되는 것은 아니다. 가 제일 인상깊음 항상 자기 피드백이 중요함
